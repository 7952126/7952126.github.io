要认真的看视频哦~

函数式编程简介 and why java8

  1. 函数式编程没有 if 之类的赋值
  2. 函数不是 java 的一种数据类型
  3. 函数式编程不会修改传入的数据
  4. 为什么要用 java8 (1) 首先就是可以写更少的代码，这样就可以写更少的 bug (2) 对并发比较有利，可以高效利用计算机多核心的能力(但是多线程我感觉不太容易啊啊啊啊啊啊啊啊)

java8 的新特性

  1. forEach(自己有用到过)
  2. 在接口中支持 default(默认方法)
  3. 支持函数式接口和 lambda 表达式
  4. Java Stream API
  5. Optional 类,可以避免空指针异常
  6. 然后在讲抽取重复的代码？然后 lambda 表达式？这个讲过的吧。。。

lambda 表达式与函数式接口

  1. lambda 表达式: 可传递匿名函数的一种方式
  2. 语法：参数列表+箭头+函数体 lambda 表达式没有方法名
  3. lambda 表达式，如果只有一条语句，可以省略{}，有多条语句就不能够省略
  • 匿名类:没有名称的类，其名称由 Java 编译器给出，一般是形如：外部类名称+$+匿名类顺序，没有名称也就是其他地方就不能引用，不能实例化，只用一次，当然也就不能有构造器。最有名的匿名类是 Comparator。
  1. lambda 能干什么:告别匿名内部类
  2. 函数式接口:只定义一个抽象方法的接口(当然，可以有很多默认接口)
  3. 函数描述符：函数式接口的抽象方法的签名，基本上就是 lambda 表达式的签名
 (T,T) -> int

  1. java8 常用的函数 (1) predicate 定义了一个名叫 test 的抽象方法。凡是涉及判断的，都可以用这个接口 (2) Consumer 接口。消费一个对象(想到了那个大嘴巴哈哈哈哈) (3) Function 接口。将 input 转化为 output，这就是函数的最基本操作嘛 (4) Supplier 接口。没有传入参数，但会返回一个泛型 T
